## 1. 背景与问题

### 1.1 现状

youlog 主题的锚点滚动分散在多个模块中协作：

| 模块 | 职责 |
|------|------|
| `youlog_lib/core/scrollAnchor.ts` | 目标解析、scrollTop 计算、滚动行为 |
| `layout/scrollLayout.ts` | 主题布局约定、`resolveYoulogAnchorScrollOffset` |
| `widgets/page-ajax/pageAjax.ts` | 首屏 hash、PJAX 导航后滚动、`pjax:anchor-navigate` |
| `widgets/toc/*` | 目录点击跳转、滚动高亮 spy offset |
| `widgets/topbar` | `--topbar-height` CSS 变量 |
| `assets/css/markdown2.css` | heading `scroll-margin-top`（原生 hash 滚动兜底） |

376becc 已将 page-ajax 首屏 hash 与 TOC 跳转偏移统一到 `resolveTocGotoHeadersHeight`，但 **偏移量的测量时机** 仍不可靠。

### 1.2 核心缺陷

1. **无统一编排**：`installToc`、`initPageAjax` 各自在 `DOMContentLoaded` 后执行，用 `requestAnimationFrame` 猜布局就绪时机。
2. **偏移量来源分散**：topbar 写 CSS 变量；mobile TOC bar 运行时 DOM 测量；CSS `scroll-margin-top` 与 JS offset 公式不一致。
3. **Solid 生命周期竞态**：小屏 `MobileToc` 在 `onMount` 后才填充 `tocItems`，`.toc-mobile-header` 晚于首屏 hash 滚动出现 → 偏移为 0。
4. **浏览器原生 hash 滚动**：JS 加载前浏览器可能已按 CSS `scroll-margin-top` 滚动，该值未计入小屏 TOC bar。
5. **布局变化无校正**：topbar 高度变化、小屏 TOC 展开/收起、PJAX 换页后 mobile TOC 重建，均可能使已滚动位置失效。
6. **调用入口过多**：page-ajax、TOC、`notifyAnchorNavigate`、未来 footnote/侧栏等均可能各自滚动，难以保证一致。

### 1.3 目标

建立 **单一锚点滚动服务 + 布局 inset 注册表 + 主题 bootstrap 编排**，使所有锚点相关场景：

- 使用同一套目标解析与偏移计算；
- 在布局稳定后再滚动（或滚动后随 inset 变化自动校正）；
- CSS 与 JS 共用 inset 变量，覆盖 JS 未执行阶段；
- 新 widget 只需注册 inset，无需改 page-ajax / TOC 内部逻辑。

### 1.4 非目标

- 不改变 everkm 锚点 id / `a.heading-anchor[name]` 协议；
- 不重构 page-ajax morph 主流程；
- 不处理站外链接、PDF 打印场景的锚点（打印另有 `installYoulogPrint`）；
- 不在 `youlog_lib` 内引入 youlog 主题专有选择器（如 `#body-main`、`#page-shell` 布局约定）或主题 CSS 路径。

### 1.5 约束：`youlog_lib` 可独立复制

`youlog_lib/` 会通过 `pull-youlog_lib` / `push-youlog_lib` **原样复制到其它主题或站点**（见 `YOULOG_LIB_README.md` §1、§3.1）。本方案须遵守：

| 原则 | 说明 |
|------|------|
| **库内零主题假设** | `youlog_lib` 只依赖 DOM/CSS 协议与可注入选项，不 import 主题 `layout/*` |
| **主题负责编排** | bootstrap 顺序、inset 组合、CSS 变量 fallback、SSR `data-*` 由宿主主题承担 |
| **可选增强** | 未传入 `AnchorScrollService` 时，widget 须降级为现有行为，保证旧项目不断裂 |
| **契约文档化** | 新增/变更 API 须同步更新对应模块 `index.ts` 文件头与 `YOULOG_LIB_README.md` |

## 2. 场景清单（须全部覆盖）

编号便于测试用例与 PR 引用。

### 2.1 进入方式

| 编号 | 场景 | 当前行为 / 风险 |
|------|------|-----------------|
| S-01 | **冷启动整页刷新**，URL 带 `#anchor` | 浏览器原生滚动 + JS `scrollToInitialHash`；小屏 TOC 未就绪时偏移错误 |
| S-02 | **冷启动**，URL 无 hash | 滚顶；无锚点问题 |
| S-03 | **PJAX 换页**，目标 URL 带 hash | `EVENT_PAGE_LOADED` 后单 rAF 滚动；mobile TOC 可能刚重建 |
| S-04 | **PJAX 换页**，目标 URL 无 hash | 滚 `#body-main` 顶部 |
| S-05 | **同页 hash-only**（点击文内 `a.heading-anchor` 或 TOC） | page-ajax 拦截或 TOC 直接滚；偏移应含当前 inset |

| 编号 | 场景 | 要点 |
|------|------|------|

| S-18 | **侧栏 drawer 打开**（stack 小屏） | 若影响 `#body-main` 可视区域，inset 不应误计 drawer |

#### 3.0.3 公共 API 契约（`youlog_lib/core`）

经 `core/index.ts` 导出，供 widget 与宿主主题使用：

```ts
/** 见 core/anchorInset.ts — 库级契约，复制 youlog_lib 时一并带走 */
export interface AnchorInsetContributor {
  id: string;
  order: number;
  measure: () => number;
  whenReady?: () => Promise<void>;
}

export interface AnchorScrollServiceOptions {
  scrollContainer: ScrollContainer | string; // 选择器或已解析容器
  articleSelector?: string;
  extraOffset?: number;
  /** 写入 documentElement 的 CSS 变量名；默认 "--anchor-inset-top" */
  insetCssVar?: string;
  /** 派发 anchor-navigate 的回调；默认使用 page-ajax 常量 */
  onNavigate?: (hash: string) => void;
}

export interface AnchorScrollService { /* 见 §3.3 */ }

export function createAnchorScrollService(
  options: AnchorScrollServiceOptions,
): AnchorScrollService;
```

**稳定性约定**：

- `AnchorInsetContributor.id` 在单页面生命周期内唯一；重复注册同 id 覆盖并 `console.warn`。
- `getOffset()` 始终同步、无副作用；适合 TOC spy 与 goto 共用。
- `whenLayoutReady()` 可多次 await，幂等。
- `scrollToHash` 返回 `boolean`（目标是否存在），**不 throw**（DOM 缺失时）。

#### 3.0.4 Widget 可选依赖契约

各 widget **不得**在内部 `createAnchorScrollService()`；须由宿主创建并注入。

| Widget | 新增选项 | 未注入时行为 |
|--------|----------|--------------|
| `initPageAjax` | `anchorScroll?: AnchorScrollService` | 沿用现有 `scrollToHash` + `resolveAnchorScrollOffset` 选项（向后兼容） |
| `installToc` | `anchorScroll?: AnchorScrollService` | 沿用 `callbackHeadersHeight` / 本地 `scrollToElement` |
| `installTopbarHeightWatcher` | `anchorScroll?: AnchorScrollService` | 仅写 `--topbar-height`，不注册 inset |

注入后 widget 职责：

- **page-ajax**：所有 hash 滚动与 `applyInitialHash` 委托 service；仍负责 `history.scrollRestoration = manual` 或与 service 协商单一职责（见 §3.7）。
- **toc**：`scrollToHeading` 委托 service；通过 `createMobileTocBarInset` 注册 contributor；删除重复的 goto offset 逻辑。
- **topbar**：通过 `createTopbarInset` 注册 contributor，并与 `--topbar-height` 同步。

#### 3.0.5 宿主主题最小接入（其它项目）

不启用 mobile TOC、无 nested scroll 的极简站点：

```ts
import { createAnchorScrollService } from "youlog_lib/core";
import { initPageAjax } from "youlog_lib/widgets/page-ajax";

const anchorScroll = createAnchorScrollService({
  scrollContainer: "body",
  extraOffset: 16,
});

initPageAjax({ scrollContainerSelector: "body", anchorScroll });
anchorScroll.applyInitialHash();
```

启用 topbar + TOC + `#body-main` 时，由主题 `layout/anchorInsets.ts` 封装 youlog 专用 contributor 注册（参考 theme-youlog `browser.ts`），**不**修改 lib 源码。

### 3.1 总览

```
browser.ts bootstrap（主题层）
    │
    ├─ 1. anchorScroll = createAnchorScrollService(...)   ← lib/core
    ├─ 2. registerYoulogAnchorInsets(anchorScroll)        ← 主题 layout/anchorInsets.ts
    ├─ 3. installTopbarHeightWatcher({ anchorScroll })
    ├─ 4. installToc({ anchorScroll, ... })
    ├─ 5. initPageAjax({ anchorScroll })
    └─ 6. anchorScroll.applyInitialHash()

AnchorScrollService（youlog_lib/core）
    ├─ InsetRegistry          各 sticky 区域注册/测量/ResizeObserver
    ├─ getOffset()            sum(insets) + extraOffset
    ├─ scrollToHash()         统一入口（解析、滚动、派发事件）
    ├─ applyInitialHash()     首屏专用：ready + 校正
    └─ onInsetChange          inset 变化时可选二次校正（仅 pending hash 会话）

scrollAnchor.ts（youlog_lib/core · 纯函数，保持不变）
    └─ computeScrollTopForElement / resolveAnchorTarget / chooseScrollBehavior
```

**原则**：`scrollAnchor.ts` 只做几何计算；**何时滚、偏移从哪来** 由 `AnchorScrollService` 决定；**哪些 inset 参与** 由宿主注册。

### 3.2 InsetRegistry（inset 注册表）

每个影响锚点顶部留白的 UI 区域作为 **inset contributor** 注册：

```ts
interface AnchorInsetContributor {
  /** 唯一 id，如 "topbar" | "mobile-toc-bar" | "body-main-header" */
  id: string;
  /** 数值越小越靠近视口顶（绘制顺序） */
  order: number;
  /** 同步测量高度（px）；无 DOM 时返回 0 */
  measure: () => number;
  /** 首次非零或稳定布局的 Promise；无 DOM 时立即 resolve */
  whenReady?: () => Promise<void>;
  /** 可选：是否参与 CSS 变量导出 */
  cssVar?: string;
}
```

**内置 contributor（youlog 主题，定义在 `layout/anchorInsets.ts`）**：

| id | 来源 | measure | whenReady |
|----|------|---------|-----------|
| `topbar` | lib：`createTopbarInset` + `installTopbarHeightWatcher` | `parseTopbarHeight()` | 首次 RO 回调 |
| `body-main-header` | **主题**：`createBodyMainHeaderInset` | `#body-main header` offsetHeight；stack=1 时返回 0 | DOM 存在即 ready |
| `mobile-toc-bar` | lib：`createMobileTocBarInset` + `installToc` | `getMobileTocBarHeight(requireSticky: false)`；viewport ≥1024 返回 0 | bar DOM 出现或确认无 TOC |

其它项目可只注册 `topbar`，或自定义 id，**无需**实现 mobile-toc-bar。

**导出 CSS 变量**（写在 `document.documentElement` 或 `#body-main`）：

```css
:root {
  --anchor-inset-top: var(--topbar-height, 0px); /* 由 JS 聚合写入 */
  --anchor-extra-offset: 10px;                 /* scrollLayout.anchorExtraOffset */
}
```

聚合公式：

```
anchorInsetTop = sum(contributor.measure()) + anchorExtraOffset
```

与现有 `resolveYoulogAnchorScrollOffset` 等价，但 **measure 时机由 Registry 统一管理**。

### 3.3 AnchorScrollService API

**路径（固定）**：

| 文件 | 层级 |
|------|------|
| `youlog_lib/core/anchorInset.ts` | 类型 + 通用 helper |
| `youlog_lib/core/anchorScrollService.ts` | 服务实现 |
| `youlog_lib/core/index.ts` | 对外 re-export |

主题层 **不得**再包一层同名 service；仅通过 `layout/anchorInsets.ts` 注册 contributor。

类型定义见 **§3.0.3**（`AnchorScrollServiceOptions`、`AnchorInsetContributor`）。补充请求类型：

```ts
interface ScrollToHashRequest {
  hash: string;
  /** initial | navigation | toc | link */
  source: AnchorScrollSource;
  behavior?: ScrollBehavior | "auto";
  /** 跳转前布局变更（如收起 mobile TOC） */
  beforeScroll?: () => void | Promise<void>;
}
```

`AnchorScrollService` 方法签名见 §3.0.3。

**`whenLayoutReady`**：

```ts
await Promise.all([
  domContentLoaded,
  ...contributors.map(c => c.whenReady?.() ?? Promise.resolve()),
]);
await nextFrame(); // 单次 rAF，确保 Solid 首帧 commit
```

**`scrollToHash` 流程**：

1. 若 `beforeScroll`，await 执行 + `nextFrame()`（必要时双 rAF，与 TOC 收起一致）；
2. `await whenLayoutReady()`（navigation 场景可跳过全量 wait，仅 remeasure）；
3. `offset = getOffset()`；
4. `scrollAnchor.scrollToHash(hash, container, { offset, behavior }, articleRoot)`；
5. `notifyNavigate()` → `pjax:anchor-navigate`。

**首屏 `applyInitialHash`**：

1. 若 URL 无 hash，return；
2. `history.scrollRestoration = manual`（仍在 initPageAjax，或服务内统一设置）；
3. 可选：**立即** `scrollToTop(0)` 取消浏览器原生 hash 跳动（或依赖 CSS scroll-padding）；
4. `await whenLayoutReady()`；
5. `scrollToHash({ hash, source: "initial" })`；
6. 注册 **一次性 inset 校正**：若 `getOffset()` 在 ready 后 500ms 内变化 > 2px，再滚一次（仅 `source === "initial"` 且用户未手动滚动）。

### 3.4 CSS 防御层（主题层，与 lib inset 变量对齐）

CSS 规则写在 **主题** `assets/css/`，不放入 `youlog_lib`。

```css
@media (max-width: 1023px) {
  #page-shell[data-has-headings="1"] #body-main {
    scroll-padding-top: var(--anchor-inset-top, calc(var(--topbar-height, 3.5rem) + 2.75rem));
  }
}

.markdown-body :is(h1, h2, h3, h4, h5, h6) {
  scroll-margin-top: var(--anchor-inset-top, var(--anchor-scroll-margin, 5rem));
}
```

说明：

- **`data-has-headings`**：JSRender 在 book 页有 heading 时输出，避免无 TOC 页多留白；
- **fallback 2.75rem**：与 `.toc-mobile-header` 典型高度一致，覆盖 JS 未执行时的 S-01 / S-09；
- JS 就绪后 `--anchor-inset-top` 覆盖 fallback，CSS 与 JS 偏移一致。

### 3.5 Bootstrap 编排（`browser.ts`，主题层）

**现顺序**（有问题）：

```
installToc → … → initPageAjax → scrollToInitialHash（内部 double rAF）
```

**目标顺序**：

```ts
// browser.ts（主题）
import { createAnchorScrollService } from "youlog_lib/core";
import { registerYoulogAnchorInsets } from "../layout/anchorInsets";
import { YOULOG_SCROLL_LAYOUT } from "../layout/scrollLayout";

const anchorScroll = createAnchorScrollService({
  scrollContainer: YOULOG_SCROLL_LAYOUT.scrollContainerSelector,
  articleSelector: YOULOG_SCROLL_LAYOUT.articleSelector,
  extraOffset: YOULOG_SCROLL_LAYOUT.anchorExtraOffset,
});

registerYoulogAnchorInsets(anchorScroll); // 主题专用 contributor
installTopbarHeightWatcher("header", { anchorScroll });
installToc({ ...YOULOG_SCROLL_LAYOUT, anchorScroll });
initPageAjax({ scrollContainerSelector: YOULOG_SCROLL_LAYOUT.scrollContainerSelector, anchorScroll });
anchorScroll.applyInitialHash();
```

`initPageAjax` 改造（lib 内）：

- 传入 `anchorScroll` 时：`scrollAfterNavigation` → `anchorScroll.scrollToHash`；**不再**内置 `scrollToInitialHash`；
- 未传入时：保留现有 `resolveAnchorScrollOffset` / `scrollToInitialHash` 路径（§3.0.4）；
- `notifyAnchorNavigate` 保留为 lib 导出，内部可委托 `anchorScroll.notifyNavigate()`。

### 3.6 TOC 改造要点（`youlog_lib/widgets/toc`）

1. **`MobileToc`**：`tocItems` 改为初始化时同步 `parseTocItems`（消除 onMount 竞态）；`onMount` 仅保留副作用。
2. **跳转**：传入 `anchorScroll` 时 `scrollToHeading` 委托 `scrollToHash`；`beforeScroll` 内 `setShowToc(false)`。
3. **inset**：新增 `createMobileTocBarInset(options)`（`tocInset.ts`），由 `installToc` 在传入 `anchorScroll` 时自动注册；未传入则保留 `callbackHeadersHeight`。
4. **`resolveTocGotoHeadersHeight`**：保留导出，标记 `@deprecated`，内部转调 inset measure 逻辑，供旧代码过渡。

### 3.7 page-ajax 改造要点（`youlog_lib/widgets/page-ajax`）

1. 新增可选 `anchorScroll?: AnchorScrollService`；
2. 有 service 时：移除本地 `scrollToPageHash` / `scrollToInitialHash`，hash 滚动与 `applyInitialHash` 外置到 bootstrap；
3. 无 service 时：保留 `376becc` 行为（`resolveAnchorScrollOffset` 等），确保其它项目未升级编排时仍可用；
4. `EVENT_ANCHOR_NAVIGATE` 仍由 service 或 page-ajax fallback 派发，语义不变。

### 3.8 事件与兼容

| 事件 | 变更 |
|------|------|
| `pjax:anchor-navigate` | 保留；由 `AnchorScrollService` 或 page-ajax fallback 派发 |
| `pjax:page-loaded` | 不变 |
| inset 调试 | 优先 `anchorScroll` 开发模式 log；**不**新增 lib 级 `youlog:*` 文档事件 |

**废弃与过渡**（跨版本）：

| 符号 | 层级 | 替代 |
|------|------|------|
| `layout/scrollLayout.resolveYoulogAnchorScrollOffset` | 主题 | `anchorScroll.getOffset()` |
| `toc.resolveTocGotoHeadersHeight` | lib | `createMobileTocBarInset` / `getOffset()` |
| `page-ajax.resolveAnchorScrollOffset` 选项 | lib | 传入 `anchorScroll` |

## 6. 验收标准

1. **一致性**：任意入口（首屏、PJAX、TOC、hash-only、popstate）滚动后，标题与 sticky 区域间距与 TOC 点击结果一致（误差 ≤ 2px）。
2. **可靠性**：小屏冷启动带 hash，连续刷新 10 次无遮挡失败。
3. **无 TOC 页**：无 heading 时 `--anchor-inset-top` 不含 mobile bar，无多余留白。
4. **可扩展**：宿主新增 sticky 区域仅需 `registerInset` 或 lib 提供的 `create*Inset`，不改 page-ajax 源码。
5. **库边界**：`youlog_lib` 内无主题 `import`；其它项目仅复制 lib + 按 §3.0.5 接线即可运行（可无 mobile TOC）。
6. **可观测**：开发模式下 service 可 `console.debug` 输出 offset 组成，便于排查。

## 7. 风险与对策

| 风险 | 对策 |
|------|------|
| 二次校正导致可见跳动 | 仅 initial hash + inset 变化 + 用户未 scroll 时校正；阈值 2px |
| Solid 仍异步插入 DOM | whenReady 绑定 mobile inset 的 RO / MutationObserver |
| CSS fallback 高度与真实 bar 不符 | 以设计 token 固定 bar 高度；JS 覆盖变量 |
| 其它项目 DOM 不同 | 仅注册适用 inset；不强制 mobile-toc-bar |
| lib 与主题职责混淆 | §3.0 归属表 + 文件头【库级/主题专用】标记 + CR 检查 |
